# x-Change — Pay Code Orchestration Platform

**x-Change** is the orchestration platform behind **Pay Code** — a rail-agnostic payment instruction framework that enables institutions to **disburse, collect, and settle** funds across incompatible rails using three transaction primitives:

- **Redeemable** — institution disburses; recipient pulls funds into any channel (bank, wallet, or cash-out agent)
- **Payable** — payer presents code; funds flow to collector via any wallet or bank
- **Settlement** — evidence-gated conditional execution via Settlement Envelope

Pay Code is to digital payments what the URL was to the internet: a **universal addressing layer** that lets the underlying infrastructure do the work.

> ⚖️ **Regulatory note:** x-Change operates **under the regulatory perimeter** of licensed banks and EMIs. It does not custody funds, transmit money, or create stored value. Pay Code functions as an instruction reference layer — not a money instrument.
